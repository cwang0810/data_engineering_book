# 中文正文风格指南

本书以工程教材为主线，正文应优先保证可信、克制、可验证。案例可以有叙事张力，但不能牺牲技术准确性和教材气质。

## 语气原则

- 用工程事实替代情绪判断，例如用“严重偏移”“显著增加成本”替代“恐怖”“惨烈”。
- 用可复现的过程描述替代网文化比喻，例如用“压力测试”“负样本挖掘”替代“毒打”“暴打”。
- 避免低俗词和人格化嘲讽，例如“屁股”“傻子”等词不进入正式章节。
- 章节结尾要服务知识衔接，不使用过度煽动或玄幻化表达。

## 常见替换

| 不建议表达 | 建议表达 |
| --- | --- |
| 毒打、暴打 | 强化训练、压力测试、对比验证 |
| 屁股 | 输入端、输出端、后续处理链路 |
| 凶萌 | 典型、具有代表性 |
| 极其恐怖 | 严重、显著、成本高 |
| 核聚变式提升 | 显著提升、效率提升 |
| 大杀四方 | 表现显著提升 |
| 终局之战 | 收束章节、关键阶段 |

## 检查方式

提交前运行：

```bash
python scripts/check_style_terms.py --baseline scripts/style_terms_baseline.json
```

如果确实需要新增某个强表达，必须先判断它是否服务技术解释；否则应改写为可验证的工程语言。
