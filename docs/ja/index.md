# 詳細目次

## 第1部：総論とインフラ

> **目標：** Data-Centric AI の体系的な認知を確立し、高性能データ処理環境の構築とコストガバナンスを完成させる。

### 第1章：LLM 時代のデータ変革

- 1.1 **スケーリングの法則からの示唆：** データ品質 > 量、「ビッグデータ」から「高品質データ」へのパラダイムシフト。
- 1.2 **LLM データライフサイクル：** 事前学習（Pre-train）→ ファインチューニング（SFT）→ 強化学習（RLHF）→ 検索拡張（RAG）。
- 1.3 **課題と機会：** 異種マルチモーダル・著作権コンプライアンス・計算コストのトレードオフ。

### 第2章：LLM データライフサイクルと品質評価フレームワーク

- 2.1 **データライフサイクル全体視点：** 収集 → クリーニング → アノテーション → イテレーションのクローズドループ体系。
- 2.2 **データ品質多次元評価：** 正確性・多様性・カバレッジ・コンプライアンスの定量化フレームワーク。
- 2.3 **品質駆動のイテレーションメカニズム：** 評価指標をデータエンジニアリング意思決定に活かす方法。

### 第3章：AI ネイティブデータスタックとコストガバナンス

- 3.1 **AI ネイティブデータスタック：**
  - ストレージ：オブジェクトストレージ（S3/MinIO）vs データレイク（Iceberg/Hudi）。
  - コンピューティング：Spark vs **Ray Data** vs **Dask** 三大分散フレームワーク比較。
  - ベクトルデータベース：Milvus / Qdrant / Weaviate / Pinecone の選定と QPS vs Recall のトレードオフ。
- 3.2 **データフォーマットと I/O 最適化：**
  - Parquet vs JSONL vs WebDataset（マルチモーダルシナリオ）。
  - 圧縮アルゴリズムと読み取りパフォーマンス最適化、GPU 学習 I/O ボトルネック最適化戦略。
- 3.3 **コストガバナンス：** 計算・ストレージ・アノテーションの全チェーンコスト計算と最適化戦略。

## 第2部：テキスト事前学習データエンジニアリング

> **目標：** 大量の非構造化テキストを処理し、モデルの言語認識基盤を構築する。

### 第4章：データソース・収集・著作権

- 4.1 **オープンソースデータセット解析：** Common Crawl・C4・RefinedWeb・The Pile・DCLM の詳細分析。
- 4.2 **高性能クローラーシステム：** `Trafilatura` 解析ライブラリの活用と分散クローラーアーキテクチャ設計。
- 4.3 **特殊データ取得：** コード（GitHub）・論文（ArXiv/S2ORC）・書籍データの抽出戦略。
- 4.4 **著作権コンプライアンス：** オープンソースライセンス識別・Robots.txt 遵守・データ認可管理。

### 第5章：クリーニング・重複除去・汚染除去

- 5.1 **ヒューリスティックフィルタリングルール：** 言語識別（FastText）・パープレキシティフィルタリング・長さと句読点分布。
- 5.2 **大規模重複除去（完全一致 vs ファジー）：**
  - **完全一致重複除去：** ハッシュ手法による完全一致ドキュメントの高速除去。
  - **ファジー重複除去：** MinHash LSH アルゴリズムの原理と分散実装。
- 5.3 **プライバシークリーニング（PII 除去）：** Presidio を使用してメール・IP・電話番号・住所を識別しマスク。
- 5.4 **ベンチマーク汚染除去：** 学習データに GSM8K・MMLU などのテストセット原題が含まれないことを確認。

### 第6章：トークン化・シリアライズ・効率的ローディング

- 6.1 **トークナイザーの原理：** BPE・WordPiece・Unigram および Byte-Level BPE の詳細解説。
- 6.2 **効率的な語彙構築：** ドメイン特化語彙拡張と中国語語彙拡張のエンジニアリング実践。
- 6.3 **データ混合（Data Mixing）：** 動的サンプリング戦略と Curriculum Learning データ配置。
- 6.4 **高性能 DataLoader：** マルチプロセスプリフェッチ・メモリマッピング・GPU 直接転送最適化。

### 第7章：データ評価・品質クローズドループ・運用イテレーション

- 7.1 **モデルベース品質スコアリング：** fastText/BERT を使用した「教科書レベル品質」スコアリング。
- 7.2 **データレシピ（Data Recipe）：** 複数ソース混合比率の実験と最適化。
- 7.3 **品質フライホイール：** 評価結果の書き戻しによるデータイテレーションクローズドループメカニズム。

## 第3部：マルチモーダルデータエンジニアリング

> **目標：** 画像・動画・音声を処理し、GPT-4V/Sora 類モデルの学習をサポートする。

### 第8章：画像テキストペアデータエンジニアリング

- 8.1 **データパラダイム：** 画像テキストペア（LAION-5B）vs インターリーブドキュメント（OBELICS/MMC4）。
- 8.2 **画像取得と前処理：** `img2dataset` 高並列ダウンロード・GPU アクセラレーションデコード（NVIDIA DALI）。
- 8.3 **マルチモーダルクリーニングパイプライン：**
  - **美的スコアリング：** CLIP-Score を使用して高美的価値の画像をフィルタリング。
  - **画像テキスト整合フィルタリング：** 説明と画像が一致しないサンプルを除去。
  - **安全性検出：** NSFW とウォーターマーク検出。

### 第9章：再キャプション付けと文書理解

- 9.1 **Alt-text の限界：** なぜ元のウェブページの説明は使用できないのか？
- 9.2 **合成キャプションファクトリー：** BLIP-2 / LLaVA / CogVLM を利用して詳細なキャプションを再生成；プロンプト戦略で粒度を制御。
- 9.3 **OCR 強化と文書理解：** 画像内テキストを抽出して説明に融合させ、文書画像理解タスクをサポート。

### 第10章：動画と音声データエンジニアリング

- 10.1 **動画処理パイプライン：** シーン分割（Scene Detection）とキーフレーム抽出戦略。
- 10.2 **動画トークン化：** 動画圧縮と離散表現。
- 10.3 **音声アライメント：** Whisper を使用した大規模 ASR および Force Alignment（タイムスタンプアライメント）。

### 第11章：クロスモーダルアライメントと融合

- 11.1 **アライメント目標：** 視覚-言語アライメントの技術パスと評価基準。
- 11.2 **クロスモーダルデータ配比戦略：** 画像テキスト/動画/音声の混合比率実験。
- 11.3 **マルチモーダルデータ品質評価：** CLIP Score・意味的一致性・対話合理性の自動化評価フレームワーク。

## 第10部：実践プロジェクト

> **目標：** 10 個のエンドツーエンドプロジェクトを通じて全書の技術ポイントを串刺しにし、実行可能なコードベースを提供する。

### プロジェクト1：Ray による分散 Mini-C4 データパイプライン構築

- **シナリオ：** Common Crawl の生データ（WARC）から高品質 Parquet データへ。
- **コア技術：** Trafilatura 解析、Ray 分散 MinHash 重複除去、KenLM 品質フィルタリング。
- **出力：** クリーン済みプレーンテキストコーパスと処理パイプライン。

### プロジェクト2：垂直ドメインエキスパート SFT（法律）

- **シナリオ：** 非構造化 PDF 文書から業界専門家ファインチューニングデータを構築。
- **コア技術：** Self-Instruct による指示構築、CoT 推論強化、データ多様性バランス。
- **出力：** `domain_expert.jsonl` 指示ファインチューニングデータセット。

### プロジェクト3：LLaVA マルチモーダル指示データファクトリー

- **シナリオ：** 画像を理解できるマルチモーダルモデルの学習。
